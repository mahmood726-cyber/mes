## Abstract

We present MES (Multiverse Evidence Synthesis), an open-access tool for conducting multiverse
meta-analysis. MES extends traditional meta-analysis by treating evidence quality, study design,
estimator choice, confidence interval method, and bias correction as dimensions in a specification
space, executing all defensible combinations and reporting robustness-classified evidence landscapes.
The default specification space comprises 648 base specifications (6 estimators × 3 CI methods ×
4 bias corrections × 3 quality filters × 3 design filters), expandable to 9,072 analyses per
dataset when leave-one-out sensitivity is included. The tool classifies each dataset into one of four
robustness tiers — ROBUST, MODERATE, FRAGILE, or UNSTABLE — based on the significance concordance
index C_sig (fraction of specifications agreeing on direction and significance), and identifies the
dominant driver of analytical variability via one-way ANOVA decomposition (eta-squared). MES is
implemented as both a Python package (mes-core) for batch research pipelines and a browser-based
single-file HTML application for interactive analysis, requiring no installation. All six estimators
have been validated against the R metafor package at tolerance 1e-4, and trim-fill against
metafor::trimfill(). The test suite comprises 126 unit tests covering all pipeline stages. MES is
freely available at GITHUB_PLACEHOLDER under an MIT license.

## Introduction

### Background

Meta-analysis is the statistical cornerstone of evidence-based medicine, pooling results from
multiple studies to produce a single authoritative estimate of treatment effects. Established
software tools — including RevMan, MetaXL, OpenMetaAnalyst, and the R package metafor — make
standard random-effects pooling accessible to systematic reviewers.1-4 However, every meta-analysis
embeds numerous analytical degrees of freedom: the choice of heterogeneity estimator, the method
for constructing confidence intervals, whether and how to correct for publication bias, which studies
to include based on quality and design, and which leave-one-out sensitivity analyses to report. Each
choice is defensible; yet each choice can materially change the pooled estimate and its statistical
significance.

### Rationale

Steegen et al. (2016) formalised the concept of the "multiverse analysis" in psychology, advocating
that researchers report results across all defensible analytical choices rather than a single
chosen path.5 In meta-analysis, this concern is particularly acute: the Fragility Atlas analysis
of 407 Cochrane meta-analyses found that conclusions were sensitive to small changes in study
inclusion and estimator choice across a substantial proportion of reviews.6 The phenomenon of
"researcher degrees of freedom" in meta-analysis has been documented empirically, yet no tool
allows systematic reviewers to execute and summarise the full specification space routinely.

Existing approaches require manual iteration across software tools or bespoke R scripting, which
is inaccessible to most clinical guideline developers and systematic reviewers. A dedicated
multiverse meta-analysis tool — one that encodes defensible specifications, executes them
automatically, and summarises the resulting evidence landscape with a principled robustness
classification — would reduce ad hoc "cherry-picking" of analytical choices and help users
distinguish genuinely robust conclusions from fragile ones.

### Aims

We describe MES, which fills this gap by providing: (1) a structured specification language for
the multiverse of defensible analytical choices; (2) automated execution of all specifications
with provenance tracking; (3) concordance-based robustness classification; (4) decomposition of
analytical variance to identify which choices drive conclusion instability; and (5) a
certification mechanism that flags whether the analysis meets quality thresholds.

## Implementation

### Architecture

MES uses a four-phase pipeline — ASSESS → EXPLORE → MAP → CERTIFY — implemented as a Python
package (mes-core) and mirrored in a standalone browser application (mes-app.html). Each phase
has a clearly defined input/output contract expressed through typed Python dataclasses, enabling
independent testing and replacement of components.

**Phase 1 — ASSESS:** Each study is annotated into a `StudyDossier` object containing its effect
size, sampling variance, design tier (1 = RCT, 2 = quasi-experimental/Mendelian randomisation,
3 = observational), risk-of-bias (RoB 2 or ROBINS-I) assessment, and participant counts. A
`BiasProfile` object is computed for the review as a whole, containing Egger's test p-value,
Begg's test p-value, and excess-significance count.

**Phase 2 — EXPLORE:** A `MESSpec` object declares the specification space — by default 648 base
specifications. The executor iterates over all combinations of estimator, CI method, bias
correction, quality filter, and design filter; applies the relevant subset of dossiers; computes
the pooled estimate; and records a `SpecResult`. When leave-one-out sensitivity is enabled, each
study is successively excluded, yielding up to 9,072 analyses for a 13-study review. Unfeasible
combinations (e.g., bias corrections requiring k ≥ 5 studies when fewer remain after quality
filtering) are skipped and counted.

**Phase 3 — MAP:** The specification results are synthesised into a `MESVerdict`. Concordance
metrics (C_dir, C_sig, C_full) quantify fraction agreement on direction, significance, and clinical
importance. Robustness classification maps C_sig to ROBUST (≥0.90), MODERATE (≥0.70), FRAGILE
(≥0.50), or UNSTABLE (<0.50). Influence decomposition (one-way ANOVA eta-squared on pooled theta
across each dimension) identifies the dominant driver of analytical variance. Fragility boundaries
identify dimension levels that flip the majority conclusion.

**Phase 4 — CERTIFY:** A TruthCert bundle records input hash, specification hash, execution
metadata, and verdict, returning PASS (error rate <5%, infeasible rate <10%), WARN (infeasible
rate 10–100%, suggesting sparse data), or REJECT (error rate >5%).

### Python Engine (mes-core)

The mes-core package (version 0.1.0) comprises 25 Python files and 1,751 lines of source code,
organised into four sub-packages:

| Sub-package | Modules | Responsibility |
|-------------|---------|----------------|
| `assess/` | `dossier_builder`, `design_classifier`, `rob_scorer`, `bias_profiler` | Study annotation and review-level bias profiling |
| `explore/` | `estimators`, `ci_methods`, `bias_corrections`, `spec_generator`, `executor` | Specification enumeration and execution |
| `map/` | `concordance`, `classifier`, `influence`, `boundaries`, `landscape`, `certifier` | Evidence landscape synthesis |
| `io/` | `csv_reader`, `rda_reader`, `exporter` | Data ingestion and export |

The top-level `pipeline.py` exposes a single `run_mes()` function that accepts a list of study
dicts and an optional `MESSpec`, returning a fully populated `MESVerdict`. Input and specification
hashes (SHA-256, first 16 hex characters) are embedded in the certification bundle for
reproducibility auditing.

Dependencies are deliberately minimal: numpy ≥1.24 and scipy ≥1.10. No meta-analysis framework
dependency is introduced; all estimators are implemented from first principles, validated against
metafor.

### Browser Application (mes-app.html)

The browser interface is a 3,318-line single-file HTML application using Plotly.js (v2.35.0) for
interactive visualisation and vanilla JavaScript for the analysis engine (porting all pipeline
stages from Python). No server-side components, installation, or API keys are required. The
application state persists in browser memory across tabs.

The interface is organised into five tabs:

1. **Data Input** — paste or type study data (study ID, yi, vi, design type, RoB assessment);
   load example datasets (BCG vaccine, robust example, fragile example, unstable example)
2. **Evidence Assessment** — review study dossiers, design tier classification, and publication
   bias profile (Egger, Begg, excess-significance)
3. **Multiverse Explorer** — configure the specification space, run the analysis, inspect the
   specification curve (sorted pooled estimates with 95% CIs) and Janus plot (volcano plot
   of theta vs. -log10 p)
4. **Evidence Landscape** — view the full verdict: significance concordance traffic light,
   conditional robustness breakdown by quality and design filter, influence sunburst chart
   (eta-squared decomposition), decision stability heatmap (C_sig by estimator × bias
   correction), and fragility boundary cards
5. **Report & Certify** — structured plain-text report with TruthCert bundle (JSON), copyable
   for methods appendices and supplementary data

Visualisations are interactive (hover, zoom, pan) and include:
- **Specification curve:** all feasible pooled estimates sorted by theta, colour-coded by
  significance; renders the full multiverse in a single panel
- **Janus plot:** theta (x-axis) vs. -log10(p) (y-axis); distinguishes direction × significance
  quadrants analogous to a volcano plot
- **Influence sunburst:** hierarchical Plotly sunburst showing eta-squared decomposition across
  all five dimensions
- **Decision stability heatmap:** C_sig by estimator (rows) × bias correction (columns) to
  reveal which analytical combinations drive instability
- **Traffic light:** single visual summary — C_sig percentage and robustness class badge
  (ROBUST/MODERATE/FRAGILE/UNSTABLE)

### Statistical Methods

**Heterogeneity estimators.** Six estimators are implemented, all validated against
metafor::rma():

| Estimator | Method | Reference |
|-----------|--------|-----------|
| FE | Fixed-effect (tau2 = 0) | Hedges & Olkin 1985 |
| DL | DerSimonian-Laird moment | DerSimonian & Laird 1986 |
| REML | Restricted maximum likelihood (Fisher scoring) | Viechtbauer 2005 |
| PM | Paule-Mandel iterative (Q* = k-1) | Paule & Mandel 1982 |
| SJ | Sidik-Jonkman one-step multiplicative | Sidik & Jonkman 2005 |
| ML | Maximum likelihood (Newton-Raphson) | Hardy & Thompson 1996 |

REML and ML use DL as the starting value and fall back to DL on non-convergence (>100 iterations).

**Confidence interval methods:**

| Method | Description |
|--------|-------------|
| Wald | Normal-approximation CI (z-critical) |
| HKSJ | Hartung-Knapp-Sidik-Jonkman (q-hat correction, t_{k-1}) |
| t-dist | t-distribution CI with k-1 df (no q-hat adjustment) |

**Bias corrections:**

| Method | Description | Minimum k |
|--------|-------------|-----------|
| none | Uncorrected pooled estimate | 1 |
| trim-fill | Duval-Tweedie L0 trim-and-fill | 5 |
| PET-PEESE | Stanley-Doucouliagos conditional regression | 3 |
| selection-model | Vevea-Hedges 3-parameter step-function selection model | 5 |

**Concordance metrics:**
- C_dir: fraction of feasible specifications agreeing on direction (negative/positive)
- C_sig: fraction agreeing on direction AND significance (primary classification criterion)
- C_full: fraction agreeing on direction, significance, AND MCID threshold (when provided)

**Influence decomposition:** One-way ANOVA eta-squared is computed for each dimension (estimator,
ci_method, bias_correction, quality_filter, design_filter) against the pooled theta, quantifying
the proportion of cross-specification variance attributable to each analytical choice.

**Prediction interval null rate:** The fraction of feasible specifications whose 95% prediction
interval includes the null value (0 on log scale), indicating the proportion of expected
future study effects that would be null even when the pooled estimate is significant.

## Use Case: BCG Vaccine Meta-Analysis

To illustrate MES, we apply it to the canonical BCG vaccine dataset (13 RCTs examining protection
against tuberculosis), first analysed by Colditz et al. (1994) and widely used in methodological
research.7

### Data

The dataset contains 13 randomised controlled trials (all design tier 1, no RoB annotation
applied), with log risk ratios (logRR) and corresponding sampling variances. The dataset is bundled
with mes-core as `data/built_in/bcg_vaccine.json`.

### Running the Analysis

```python
from mes_core.pipeline import run_mes
import json

with open("data/built_in/bcg_vaccine.json") as f:
    studies = json.load(f)

verdict = run_mes(studies)
print(verdict.overall_class)   # FRAGILE
print(verdict.overall_c_sig)   # 0.6563
```

With default settings, MES generates 9,072 specifications (648 base × 14 LOO iterations). After
excluding infeasible combinations (bias corrections requiring k ≥ 5 when quality/design filters
reduce k below threshold), a feasible set of specifications is executed. The analysis produces a
WARN certification because the infeasible rate exceeds 10%, reflecting that some quality-filter
strata produce very small k in a 13-study review.

### Verdict

| Metric | Value |
|--------|-------|
| Overall class | FRAGILE |
| C_sig | 0.6563 |
| Dominant dimension | bias_correction (eta2 = 0.753) |
| Estimator eta2 | 0.051 |
| CI method eta2 | 0.000 |
| Prediction interval null rate | 0.931 |
| Certification | WARN |

The BCG dataset is FRAGILE (C_sig = 0.66), meaning that 34% of defensible specifications yield a
different direction/significance outcome. The dominant driver is bias correction (eta2 = 0.753):
when bias corrections are applied (trim-fill, PET-PEESE, selection model), the pooled log RR
shifts substantially toward the null relative to the uncorrected estimate. Estimators contribute
only 5.1% of the variance; CI method choice has no impact.

This pattern is interpretively important: the widely-reported conclusion that BCG vaccination
reduces TB risk (~50% RR reduction) is driven by the raw pooled estimate. Under bias-corrected
specifications, the effect attenuates, reflecting the known funnel plot asymmetry in this dataset.8
The prediction interval null rate of 93.1% further indicates that, even when the mean effect is
significant, the expected effect in any given future study is likely to include zero, consistent
with the substantial heterogeneity (I2 ≈ 92%) in this dataset.

### Conditional Robustness

When conditioning on quality filter:

| Quality filter | C_sig | Class |
|----------------|-------|-------|
| all | 0.626 | FRAGILE |
| exclude-high-rob | 0.626 | FRAGILE |
| low-rob-only | 0.626 | FRAGILE |

With only 13 RCTs (all tier 1), quality filtering does not change the conclusion class for this
dataset; the fragility arises entirely from bias correction choice.

## Validation

### R Parity

All six estimators were validated against metafor::rma() using the BCG vaccine dataset (k = 13,
logRR scale). The validation script (`validation/r_parity/validate.R`) reads the same
`data/built_in/bcg_vaccine.json` file used by the Python tests to ensure identical data.

R-validated estimator results for the BCG dataset:

| Estimator | Python theta | R theta | Python tau2 | R tau2 | Max abs delta |
|-----------|-------------|---------|-------------|--------|---------------|
| FE | -0.4321 | -0.4321 | 0.0000 | 0.0000 | <1e-6 |
| DL | -0.7141 | -0.7141 | 0.3084 | 0.3084 | <1e-5 |
| REML | -0.7145 | -0.7145 | 0.3133 | 0.3133 | <1e-4 |
| PM | -0.7150 | -0.7150 | 0.3181 | 0.3181 | <1e-4 |
| SJ | -0.7172 | -0.7172 | 0.3455 | 0.3455 | <1e-4 |
| ML | -0.7112 | -0.7112 | 0.2800 | 0.2800 | <1e-4 |

Trim-and-fill was validated against metafor::trimfill() (DL estimator) with agreement within
1e-3 on the adjusted pooled estimate. Egger's test p-value was validated against
metafor::regtest() within 1e-6.

The tolerance for REML, PM, SJ, and ML (1e-4) reflects genuine numerical differences in
optimiser implementation between the Python Fisher-scoring / iterative solvers and the Fortran
routines in metafor; all differences are below any clinically or statistically meaningful
threshold.

### Test Suite

| test_concordance | C_dir, C_sig, C_full | empty list, k=1 |
| test_classifier | robustness tier thresholds | boundary values |
| test_influence | eta-squared decomposition | uniform theta |
| test_boundaries | flip detection | all-agree, boundary cases |
| test_landscape | synthesize_verdict integration | BCG, robust, fragile |
| test_certifier | PASS/WARN/REJECT thresholds | error/infeasible rates |
| test_pipeline | run_mes end-to-end | BCG, robust, fragile, unstable |
| test_models | dataclass field types | edge values |

| test_io | CSV reader, exporter | round-trip |

All 126 tests pass (15.9 s on a standard laptop).

### Empirical Validation

The batch runner (`validation/batch_run.py`) was applied to 403 Cochrane systematic reviews from
the Fragility Atlas dataset, processing each review through the full MES pipeline. Zero pipeline
errors were recorded across 403 reviews. The distribution of overall robustness classes across
this corpus is consistent with prior work reporting widespread fragility in Cochrane meta-analyses.6
